A standard interface for tokens.

The following standard allows for the implementation of a standard API for tokens within smart contracts.
This standard provides basic functionality to transfer tokens, as well as allow tokens to be approved so they can be spent by another on-chain third party.

A standard interface allows any tokens on Ethereum to be re-used by other applications: from wallets to decentralized exchanges.

- **totalSupply**: The return value `supply` must match the current `_totalSupply`.
- **transfer**: 
  - If the sender is transferring to themselves, their balance remains the same. If they are transferring to another address, that address's balance increases by `_value`.

  - The success condition checks for successful transfers or handles zero-value transfers.
- **transferFrom**: 
  - Similar to `transfer`, it checks balances for both the sender and the receiver, and it also decrements the allowance for the spender.
  - It checks for successful execution or handles zero-value transfers.
- **approve**: The allowance for the spender is set to `_value`, and it checks for successful execution or handles zero-value approvals.
- **balanceOf**: The returned balance must match the balance stored in `_balances`.
- **allowance**: The returned remaining allowance must match the allowance stored in `_allowed`. 

These postconditions ensure that the functions behave according to the ERC20 standard and maintain consistency across the contract.
