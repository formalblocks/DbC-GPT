### Explanation of Postconditions:
1. **totalSupply()**: The return value `supply` must equal the state variable `_totalSupply`.
  
2. **transfer()**: 
   - If the transfer is successful, the balance of the sender decreases by `_value` and the balance of `_to` increases by `_value` unless `_to` is the same as the sender (self-transfer).
   - If the transfer is unsuccessful, the balances remain unchanged.

3. **transferFrom()**: 
   - Similar to `transfer()`, but it also checks that the allowance for `_from` has been decreased by `_value` upon a successful transfer.
   - The same conditions apply for self-transfer and failure cases.

4. **approve()**: 
   - If the approval is successful, the allowance for `_spender` is set to `_value`.
   - If `_value` is zero, the allowance should remain unchanged (allowance can be set to zero to avoid race conditions).
   - If the approval is unsuccessful, the allowance remains unchanged.

5. **balanceOf()**: The returned `balance` must equal the balance of the specified owner.

6. **allowance()**: The returned `remaining` must equal the allowance that `_spender` has from `_owner`.
