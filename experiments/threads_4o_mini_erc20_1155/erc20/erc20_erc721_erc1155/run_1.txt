This standard provides basic functionality to transfer tokens, as well as allow tokens to be approved so they can be spent by another on-chain third party.

A standard interface allows any tokens on Ethereum to be re-used by other applications: from wallets to decentralized exchanges.

1. **`totalSupply`**: Ensures that the returned supply matches the total supply of tokens stored in the contract.

2. **`transfer`**: 
   - Verifies that the sender's balance is correctly updated (decreased) when sending tokens to another address, except when sending to themselves.
   - Ensures that the recipient's balance is correctly updated (increased).
   - Checks that the `Transfer` event is emitted.

3. **`transferFrom`**: 
   - Similar to `transfer`, it ensures that balances are updated correctly for both the sender and the recipient.
   - It checks the allowance for the `_from` account is decremented correctly.
   - Ensures the `Transfer` event is emitted.

4. **`approve`**: 
   - Checks that the allowance for the spender is set correctly and that the `Approval` event is emitted.

5. **`balanceOf`**: 
   - Validates that the returned balance for the specified owner matches the stored balance.

6. **`allowance`**: 
   - Ensures that the returned allowance matches the stored allowance for the specified owner and spender.

These postconditions help ensure that the contract behaves according to the ERC20 standard and that the state updates are consistent and predictable.
