## Simple Summary

A standard interface for tokens.


## Abstract

The following standard allows for the implementation of a standard API for tokens within smart contracts.
This standard provides basic functionality to transfer tokens, as well as allow tokens to be approved so they can be spent by another on-chain third party.


## Motivation

A standard interface allows any tokens on Ethereum to be re-used by other applications: from wallets to decentralized exchanges.

    mapping (address => uint) _balances;
    mapping (address => mapping (address => uint)) _allowed;
    uint public _totalSupply;

    * *Note* Transfers of 0 values MUST be treated as normal transfers and fire the `Transfer` event.
    */

    /// @notice postcondition _balances[msg.sender] >= _value || !success
    /// @notice postcondition success == (_balances[msg.sender] >= _value && _to != address(0))
    /// @notice postcondition msg.sender == _to || success || _value == 0
    function transfer(address _to, uint256 _value) public returns (bool success);

    * Transfers `_value` amount of tokens from address `_from` to address `_to`, and MUST fire the `Transfer` event.
    * The function SHOULD `throw` unless the `_from` account has deliberately authorized the sender of the message via some mechanism.
    * *Note* Transfers of 0 values MUST be treated as normal transfers and fire the `Transfer` event.
    */

    /// @notice postcondition _balances[_to] == __verifier_old_uint(_balances[_to]) + _value || !success
    /// @notice postcondition _balances[_from] >= _value || !success
    /// @notice postcondition _allowed[_from][msg.sender] >= _value || !success
    /// @notice postcondition success == (_balances[_from] >= _value && _allowed[_from][msg.sender] >= _value && _to != address(0))
    /// @notice postcondition _from == _to || success || _value == 0

    * Allows `_spender` to withdraw from your account multiple times, up to the `_value` amount.
    * If this function is called again, it overwrites the current allowance with `_value`.
    */

    /// @notice postcondition success == true

    * Returns the account balance of another account with address `_owner`.

1. **totalSupply**: The postcondition ensures that the returned `supply` is equal to `_totalSupply`.

2. **transfer**:
   - The sender's balance should decrease by `_value` if the operation is successful.
   - The recipient's balance should increase by `_value` if successful.
   - Checks that the sender has enough balance.
   - Ensures the operation succeeds only if the sender's balance covers the `_value` and the recipient address is valid.
   - If `_value` is zero or `_to` is the same as `msg.sender`, it is considered a successful operation in terms of requirement.

3. **transferFrom**:
   - Similar to `transfer`: adjusts balances and checks authorization (`_allowed`).
   - Ensures the transfer's success is dependent on both balance and allowance adequacy, and a valid recipient.

4. **approve**:
   - Sets the allowance for `_spender` to `_value`.
   - Ensures the function always returns true as there is no failure mode defined in ERC20 for `approve`.

5. **balanceOf**: Simply relates the returned balance directly to the `_balances` mapping for `_owner`.

6. **allowance**: Relates the returned remaining allowance directly to `_allowed[_owner][_spender]`. 

These annotations ensure formal verification aligns with the intended behavior and constraints of the ERC20 standard.
